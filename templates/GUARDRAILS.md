# GUARDRAILS.md — invariant ledger

> Generated by `guardrails:init-guards`. This is the project's **definition of done, semantic layer** —
> the recurring defect classes and the automatic guard that kills each. The type-check/lint/test gate is
> orthogonal to these: code can compile and pass while it leaks data or swallows an error. Every recurring
> defect class becomes a check that FAILS at write time. Guard strength ladder: 🔒 lint > 🧪 test > 👁 required reviewer.
>
> Rule: every bug fix kills the CLASS (grep siblings, fix all), strengthens the guard one rung, and updates
> its ledger line — same commit. A class with no guard reappears later as a "new" bug.
